解决ubuntu更新17.10后开机进不了桌面的问题

发表于 2018-03-10 | 分类于 configure |
最近解决了一个比较坑的问题，从ubuntu16.04直接用upgrade更新到17.10之后，发现开机之后进不了桌面，折腾了一下最终解决了。

【转载请注明出处】chenrudan.github.io

最近解决了一个比较坑的问题，从ubuntu16.04直接用upgrade更新到17.10之后，发现开不了机，一直停留在在ubuntu的缓冲界面，类似下图1，无论等多久都进不去桌面，折腾了一下目前看来正常的功能都能使用，还没有遇到什么奇怪的状况。

图1 ubuntu缓冲界面

进入图2中的advanced options for ubuntu

图2 advanced options

随便选择一个kernel的恢复模式

图3 recovery mode

图4和图5选择修复破损包

图4 Repair broken packages

图5

修复期间，在图5中的步骤中不知为啥卡住了，出门剪了个头发回来还没setting好，这里也不知道是干吗的，直接ctrl+c，这个包打断之后，后面的包继续安装。

图6 remount

安装完之后就可以进入桌面了，此时我的桌面变成了分辨率很奇怪的情况，且报了下面图7的错。

图7 urfkill

暂时没管，直接重启电脑，登录之后又报了图7的错，且此时打不开终端，鉴于这个情况，继续简单粗暴的更新安装包。

图8 Software Updater

图9 Install Now

经过上述的一些骚操作，反正我的电脑能够正常登进桌面和使用了，系统也是17.10，虽然过程比较简单粗暴，很多地方也不知道为什么，但是确实给折腾好了，真是个神奇的世界。
